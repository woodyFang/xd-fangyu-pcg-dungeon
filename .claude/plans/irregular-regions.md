# 添加异形区域支持计划

## 目标
让“区域”不再只能是矩形/椭圆/八边形，而是可以一键切换为不规则的异形区域：边缘有凹凸、缺口和块状扩展，同时仍能参与现有自动分离、连线、门槽、BFS 连通校验、装饰和 3D 渲染。

## 现状理解
- 区域数据主要在 `src/main.js` 的 `room.shape` 字段中流转。
- 当前生成支持 `rect`、`ellipse`、`oct`，栅格化入口在 `tryGenerate()` 的房间 rasterize 段。
- 2D 编辑器使用 `editor.rooms` 保存手工区域，右键菜单已有“添加区域/删除/锁定/连接/角色”等功能。
- 编辑器命中、绘制、3D 选中轮廓目前基本按矩形包围盒处理；真实房间点击命中对 `ellipse/oct` 有额外判断。

## 实现方案
1. **新增异形 shape 类型**
   - 在房间 shape 中增加 `organic` 或 `irregular` 类型。
   - 随机生成时给一部分中大型区域使用异形类型，避免所有区域都变碎。
   - 手动添加区域默认仍保持矩形，避免误操作；通过菜单切换为异形。

2. **封装统一的房间形状判断函数**
   - 新增 `roomShapeMask(room, x, y)` 或类似函数，用于判断某格是否属于房间内部。
   - 复用到：
     - 房间栅格化
     - 3D 场景点击选房间
     - 2D 编辑器绘制预览
   - 对异形区域使用确定性噪声/哈希，基于 `room.id/sourceId/seed` 生成稳定边缘：同一种子、同一编辑布局结果稳定。

3. **异形区域栅格化策略**
   - 以矩形包围盒为基础，先生成“核心区域”，保证中心和门槽附近可通行。
   - 在边缘区域用多频噪声或分块哈希做凹凸裁剪，形成不规则轮廓。
   - 加保护规则：
     - 中心附近始终保留，避免起点/终点/宝箱中心落空。
     - 至少保留一定比例面积，太碎则退回八边形或矩形。
     - 门槽和走廊 carving 仍可强制打通，现有 BFS 负责最终连通校验。

4. **编辑器交互**
   - 在区域右键菜单增加：`切换异形`。
   - 对选中区域执行 `rect -> organic -> ellipse -> oct -> rect`，或只做 `rect/organic` 切换；建议先做循环切换，方便测试已有形状。
   - 2D 编辑器绘制时对异形区域画出近似轮廓，而不是只画矩形框；选中/缩放手柄仍沿包围盒，便于编辑。

5. **3D 选中和轮廓**
   - 点击房间时使用同一形状判断函数，避免点击异形缺口也选中房间。
   - 选中轮廓第一版仍可保留包围盒高亮，降低复杂度；如果时间允许再用边界格生成折线轮廓。

6. **验证**
   - 运行 `npm run build` 确认打包通过。
   - 如需要视觉验证，再启动 Vite 预览并用浏览器检查：
     - 随机生成中能看到异形区域。
     - 右键区域可切换异形。
     - 异形区域仍能连通、装饰和选中。

## 主要修改文件
- `src/main.js`
  - shape 生成/记录
  - 栅格化形状判断
  - 编辑器菜单和绘制
  - 点击命中/同步逻辑
- `index.html`
  - 右键菜单新增“切换形状/切换异形”按钮
- `src/ui/styles.css`
  - 如菜单无需特殊样式，可能不改；仅在需要形状标签样式时微调

## 风险与控制
- 异形边缘可能导致门口落在缺口：通过门口 carving 和中心/边缘保护降低风险。
- 太碎会影响装饰摆放：用保底面积和核心区域保证。
- 编辑器命中若完全按异形判断会难以拖动缺口区域：编辑器仍用包围盒操作，视觉上画异形轮廓。
